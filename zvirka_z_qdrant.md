# Звірка бази даних Qdrant з посиланнями на судові рішення

## Загальний алгоритм звірки:
1. **Збір еталонних даних (з файлу)**: Читається ваш вихідний файл із посиланнями (наприклад, CSV, JSON або звичайний текстовий файл) і з нього витягується повний список унікальних ідентифікаторів (це можуть бути самі посилання, номери справ або спеціальні ID).
2. **Збір наявних даних (з бази)**: Робиться запит до вашої бази даних (через Prisma або безпосередньо в Qdrant/PostgreSQL) для отримання списку всіх ідентифікаторів, які вже успішно збережені.
3. **Порівняння (знаходження різниці)**: Програмно (наприклад, за допомогою `Set` в JavaScript) порівнюються ці два списки. Знаходиться різниця: усі посилання, які є в першому списку (файл), але відсутні в другому (база).
4. **Результат**: Формується новий звіт (або масив даних), який містить лише ті рішення, що не завантажились. 

## Як це можна буде реалізувати технічно (на прикладі стеку з Node.js та Prisma/Qdrant):
Це можна реалізувати у вигляді невеликого скрипта (схожого на скрипт імпорту `ingest-kcs.mjs`), який робитиме наступне:

```javascript
// ПРИКЛАД ЛОГІКИ ДЛЯ МАЙБУТНЬОГО СКРИПТА
import fs from 'fs';
import { PrismaClient } from '@prisma/client';
const prisma = new PrismaClient();

async function checkMissingDecisions() {
  // 1. Читаємо всі посилання з вашого файлу
  const allLinksFromFile = [/* ... логіка читання вашого файлу ... */];
  
  // 2. Отримуємо всі збережені посилання з бази даних
  const savedRecords = await prisma.decision.findMany({
    select: { url: true } // беремо тільки поле з посиланням для швидкості
  });
  const savedLinks = new Set(savedRecords.map(record => record.url));

  // 3. Знаходимо ті, яких немає в базі
  const missingLinks = allLinksFromFile.filter(link => !savedLinks.has(link));

  // 4. Виводимо результат
  console.log(`Знайдено ${missingLinks.length} незавантажених рішень.`);
  
  // (Опціонально) Зберігаємо їх у новий файл для повторного завантаження
  fs.writeFileSync('missing_decisions.txt', missingLinks.join('\n'));
}

// Виклик функції (з обробкою помилок)
checkMissingDecisions().catch(console.error);
```

## Переваги такого підходу:
- **Швидкість**: Порівняння десятків тисяч записів у пам'яті (через `Set` або хеш-таблиці) займає частки секунди.
- **Точність**: Ви точно знатимете, які саме файли випали через помилки мережі (timeout), обмеження API (rate limits) або інші збої.
- **Автоматизація дозавантаження**: Цей список "незавантажених" можна буде відразу передати на вхід іншому скрипту, який завантажить лише їх, не чіпаючи те, що вже є в базі.

Після завершення поточного завантаження або при необхідності перевірити повноту бази, цей підхід повністю сумісний з поточною архітектурою проєкту і дозволить легко виявити пропущені документи.
